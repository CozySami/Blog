## 5.2 生成式框架模型的基石

生成式框架通常有两种核心架构方式：
- **Transformer**：基于自回归的 Token 生成序列。
- **Diffusion**：基于迭代去噪的方式，从噪声中恢复数据。
这两种框架并非互斥，甚至可以互补协同。

#### Transformer 架构

Transformer 架构具有以下核心优势：
- **自注意力机制**：天然地建模长序列关系。
- **并行化训练**：使其能够非常高效地训练模型。
- **架构统一性与可扩展性**：可以简单地堆叠层来提升模型能力，这正是LLM 所谓的 **Scaling Law** 的。

##### 注意力机制

注意力机制的本质是让模型在预测下一个物品时，能够**动态地、选择性地关注历史行为**。这不同于 RNN 的更依赖前一个位置的顺序处理，它允许模型关注序列中的任意位置。

注意力机制的核心是 Query-Key-Value (QKV) 计算过程。其中：
- **Q (Query)**：表示当前位置想查询的信息。
- **K (Key)**：序列中各个位置提供的信息索引。
- **V (Value)**：序列中每个位置实际包含的内容。

当我们知道哪些位置重要之后，需要聚合的就是对应的 V。注意力权重的计算过程可以形式化为：

\[
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V
\]

其中：
- \( QK^T \) 计算 Query 和 Key 的匹配度（内积越大越相关）。
- \( \sqrt{d_k} \) 是一个缩放因子，用于稳定梯度。
- 得到的权重矩阵 \( A \) 与 \( V \) 相乘，即得到**加权聚合后的序列表示**。

在生成式推荐系统中，同样会用到**多头注意力（Multi-Head Attention）**。其核心思想是让不同的“头”学习到不同方面的因素（例如价格、品牌、功能），从而增强模型的表征能力。

#### 位置编码与序列建模

普通的注意力机制对序列中元素的**顺序不敏感**（item 序列 [1, 2, 3] 和 [3, 2, 1] 的结果相同）。因此，需要引入**位置编码**来打破这种对称性。

- **绝对位置编码**：为每个位置分配一个固定的编码向量。
- **相对位置编码**：编码元素之间的相对距离。新方法往往采用相对位置编码，因为它具有**更强的泛化能力**，能更自然地处理变长序列。

####  时间序列建模

在推荐场景中，位置编码需要特别考虑**时间的特殊性**，因为时间有不同的颗粒度。有两种主要的编码方式：
1.  **引入绝对时间戳信息**。
2.  **采用相对时间位置编码**，即编码位置 \( p \) 和 \( t \) 之间的时间间隔。

通常，模型会为不同的时间颗粒度维度引入单独的编码，学习一个关于**时间间隔的 Embedding 表示**。选择合适的颗粒度（如短视频的秒级、电商推荐的跨天级）对模型性能至关重要。

####  Transformer的两种主要架构

- **Encoder-Decoder 架构**：包含编码器和解码器两部分。
- **Decoder-only 架构**：仅包含解码器部分。

在双塔（Encoder-Decoder）架构中：
- **Encoder** 使用**双向自注意力**，可并行编码整个输入序列。
- **Decoder** 使用**因果自注意力（Causal Self-Attention）** 和**交叉注意力（Cross-Attention）** 来生成输出。

在单塔（Decoder-only）模型中，则统一使用**因果自注意力**。

#### 两种架构的核心区别

| 对比维度 | Encoder-Decoder 架构 | Decoder-only 架构 |
| :--- | :--- | :--- |
| **注意力机制** | 至少三种（Encoder自注意、Decoder因果注意、交叉注意） | 一种统一的因果注意力 |
| **参数分配** | 参数分布在 Encoder、Decoder、交叉注意力模块 | 参数全部集中在 Decoder |
| **计算模式** | Encoder并行编码，Decoder自回归生成 | 完全自回归生成 |
| **序列组织** | 输入和输出序列分离 | 输入和输出拼接为单一序列 |

#### 架构选择与优劣分析

- **Encoder-Decoder 架构的优势**：
    - 适合**理解与生成分离**的任务，或输入/输出信息模式差异大的场景。
    - 在异步场景下，可对输入进行预编码和缓存，**减少在线推理计算量**。

- **Encoder-Decoder 架构的劣势**：
    - **效率与可扩展性**：三组独立的注意力机制计算开销大，参数分散，难以扩展。

- **Decoder-only 架构的优势**：
    1.  **参数效率高**：参数集中，易于扩展。
    2.  **工程实现简单**：只有一种注意力矩阵。
    3.  **与 LLM 生态兼容**：可沿用成熟的大语言模型训练与优化技术。
    4.  **灵活性**：通过构造不同的**掩码矩阵（Masking Matrix）**（如 session-level masking, task-level masking, bidirectional prefixing），即可控制信息流动，无需改变模型结构。

- **Decoder-only 架构的劣势**：
    - **单向建模约束**：只能从左到右进行建模（但在推荐系统的序列续写任务中，这不一定是缺点）。
    - **上下文长度压力**：需要将整个用户历史作为上下文输入，导致序列变长，计算开销增大。

##### 实际应用中的选择考量

1.  **任务类型**：
    *   若任务包含“理解用户”和“生成推荐”两个明显过程，或输入/输出模式差异大 → 优先选择 Encoder-Decoder。
    *   若是纯粹的序列续写（推荐生成）任务 → 优先选择 Decoder-only。

2.  **数据与模型规模**：
    *   数据量非常大，支持大规模训练（如 10B+ 参数） → Decoder-only 扩展性更好。
    *   数据有限，模型规模较小（如 1B 以下） → Encoder-Decoder 可能更易优化和稳定训练。

3.  **部署与推理**：
    *   Encoder-Decoder 适合异步缓存优化, 线下预计算，送线上做推理。
    *   Decoder-only 凭借成熟的端到端优化，也能实现毫秒级的在线推理。

##### 一个技巧：因果掩码
LLM 训练过程 往往采用 因果掩码（Causal Masking ) 。在训练时，通过将未来位置的注意力权重设置为负无穷（`-inf`）来实现掩码，从而确保自回归特性。
\[
M_{ij} =
\begin{cases}
0, & \text{if } i \geq j \quad \text{(允许关注当前位置及之前的位置)} \\
-\infty, & \text{if } i < j \quad \text{(屏蔽未来的位置)}
\end{cases}
\]
Attention matrix 计算变成
\[
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}} + M\right) V
\]


### Diffusion Model

Diffusion Model 通过**迭代去噪**的方式从随机噪声中恢复数据，其生成逻辑与自回归模型不同。

根据操作空间的不同，主要有两种方式：
1.  **原始数据空间（Raw Space）扩散**：直接在原始数据维度上进行去噪。
2.  **潜空间（Latent Space）扩散**：既Stable Diffusion 原理：先将数据压缩到低维潜空间再进行去噪。**实际应用中更为常用**，因为计算效率更高。

#### Diffusion Model 在推荐系统中的应用

1.  **特征增强与表示学习**：利用其生成能力增强用户/物品的隐式表征。
2.  **数据生成与推荐**：直接生成推荐序列。与自回归模型不同，其去噪过程是**并行化的**，不受严格序列约束。
3.  **模型融合的桥梁**：作为融合多模态信息（文本、图像）的桥梁，在统一的潜空间中根据条件扩散生成结果，实现**跨模态语义对齐**。
4.  **协同过滤图构造**：将扩散过程引入用户-物品交互图的节点表示中，以捕捉更高级的图信号。

#### Diffusion Model 的主要缺点
- **推理延迟高**：由于需要多步迭代采样，导致**在线推理延迟较高**。
