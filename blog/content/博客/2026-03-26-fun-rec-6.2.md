## 6.2 GenRank架构优化 读书笔记

[原文链接](https://datawhalechina.github.io/fun-rec/chapter_6_scaling/2.gen_rank.html)

### 6.2.1 逐本溯源：什么才是本质？

小红书团队做了系统性实验，证明自回归其实并不是一个可选机制。这里的可选机制指的是用 causal mask 训练，确保第 i 个 token 只能看到之前的信息进行计算。团队使用了候选物品，历史位置不参与，这有点类似于大模型的监督微调，而不是预训练。

那么自回归是不是一个可选的 trick？他们做了两个实验：

1. 历史位置与当前为 loss，这使得自回归变成一个可选技巧，更多的监督信号进入，实际上 AUC 大幅下降。
2. 在历史位置中使用全可见 mask，允许双向 attention，AUC 同样下降，而且模型越大下降越大。

这说明全可见 mask 完全破坏了归纳偏置，破坏了因果性。这两个实验得出同一个结论：自回归机制是推荐模型的本质特征，必须通过架构进行约束，形成有益的归纳偏置，让模型学习因果结构。

另外，样本的组织方式影响较小。传统的 DLRM 训练中，每个样本对应一次交互。HSTU 采用 user level 训练，将每个用户的多次交互组成序列。实际证明，保持序列化组织，只在最后位置计算 loss，性能几乎没有下降。

这意味着，推荐的有效性来自于自回归机制，而并非训练范式的细节。

序列建模方面的实验证明，大部分历史聚合特征的价值大幅降低，因为模型的序列建模能力能够自动学会这些统计信息。但实时特征仍然非常重要，它们捕捉了窗口外的新信息。

那么问题变成了：如何保持核心优势但提高资源效率？新架构的第一个方向是 **Action Orientation**。

### 6.2.2 Action-Oriented：重新理解任务本质

我们要解决的问题是：给定用户历史和候选物品，我们真正要预测的其实是用户的反馈行为 \(a\)，而并非物品本身。在排序任务中，物品只是上下文（context），而行为才是真正要预测的目标。这意味着，将历史物品和行为平等对待并非必要。

**具体做法**：

- 将物品（Item）嵌入和行为（Action）嵌入在同一空间中直接融合，即相加。
- 对于候选物品，使用特殊的 `Mask` 占位符和 Action Embedding。

每个 token 的表示为：

\[ e_i = \varphi(x_i) + \phi(a_i) \]

**好处**：

- 序列长度直接减半。
- Attention 计算量减少约 75%。
- 线性投影参数减少 50%。
- 内存占用减少 50%。
- K/V Cache 大小减半。
- 这一改动可以实现约 78% 的训练加速。

这意味着，加法操作允许物品和行为的表征在向量空间中直接对齐。对于任何一个维度，如果这个维度既表示物品的某个特征，又表示行为的某个特征（即共享的语义维度），那么这个维度的表征可能会被加强。

如果这个维度只与物品属性相关，则行为在这个维度的值会接近于0。而如果这个维度只与行为有关，跟物品属性无关，则物品在这个维度上的表征会接近于0。

Action Orientation 还带来了另一个灵活的 Mask 设计。因为排序需要对一系列候选进行评分，每个候选需要保持独立，但能够看到用户的完整信息。GenRank 采用了特殊的 Mask 设计：通过使用 Causal Mask，候选的 Attention 可以看到全部历史，但候选之间彼此屏蔽。这样既保证了独立性，也为将来的多任务学习（如序列推荐）留下了空间。

### 6.2.3 位置与时间：该学习什么，该编码什么？

HSTU 使用了注意力偏置（Attention Bias）机制，在 Attention Score 中加入可学习的 Bias。这个 Bias 是学习位置差异和时序差异的关系，但这个维度非常大，计算和储存的开销是 \(O(N^2)\)。那么 GenRank 采取的是用现有的 Embedding 编码绝对信息，并用无参数的 Bias 编码相对信息。

GenRank 引入了三种轻量级的 Embedding：

1. **Position Embedding**：同一个请求中的候选位置共享位置索引。
2. **Request Index Embedding**：捕捉用户行为的 burst 模式，区分同一 session 内和跨 session 的交互。
3. **Pre-Request Time Embedding**：编码当前交互和上一次请求的时间间隔，实现自适应衰减。

位置编码与请求的长度有关。请求索引编码是一种基于行为发生时间的"burst"模式，即在同一个 Session 之内，使用同一个编码。这可以告诉模型哪些交互是在同一个 Session 之内发生的，是连续的，而哪些交互是有边界的，是跨 Session 的。

第三个是请求间时间编码，编码当前交互和上一次交互的时间间隔。这个时间间隔实现了自适应衰减。对于高频用户来说，短时间间隔就有意义；而对于低频用户来说，几个小时间隔并不算什么。也就是说，当当前编码和上一次请求的时间间隔超过一定范围之后，编码将使用 bucket 方式（查表）得到 embedding。

三种 embeddings 加到 token 表示：

\[ e*i^{(p,t)} = \varphi(x_i) + \phi(a_i) + E*{pe,i} + E*{ri,i} + E*{rt,i} \]

最后，GenRank 采用了 **ALiBi (Attention with Linear Biases)** 方法，就是在 Attention 中给距离相对较远的 Key 施加了惩罚，惩罚与距离成正比：

\[ \text{bias}\_{i,j} = -m_p \cdot (p_i - p_j) - m_t \cdot \text{bucket}(t_i - t_j) \]

该方法有三个优势：距离越远影响越小、无参数、而且是可融合的。

这一设计反映了一个特征：用参数编码复杂的非线性模式，用简单规则来编码普适的线性模式。采用了 Action Orientation 加速 78%，新的 Position 和 Time Bias 额外可以再加 25%，结合后可以达到加速 94.8%，同时指标还略微升高，这意味着好的归纳偏置设计比纯参数容量更重要。

Action Orientation 的创新点在于将行为作为预测目标，而物品作为上下文，使得序列长度减半。

位置编码与时间编码的重构体现了一个原则：对于距离衰减有强先验的规律，用编码来表征比从数据中学习更高效。也就是说，我们把这种对于距离衰减的规律进行直接编码，而不依赖于从数据中学习。并且在设计过程中做出了一个非常重要的选择，就是保持了任务目标的纯粹性，保证了自回归方式。这意味着不能使用任何未来的信息。同时，这要求在训练框架和推理中必须放弃传统的、基于交叉特征的方法。

其实，如果是 GenRank 这种自回归的模式，是没有办法或很难计算精心设计的交叉特征的，所以只能放弃。这其实是强迫人们放弃这种传统的建模方法。

那么最后一个问题：这种用户粒度的建模效率优势，是否必须完全放弃？能否在保持样本聚合和计算效率的同时，仍然支持灵活的建模方式？基于二者的结合，是 MTGR 所解决的问题，它提供一种混合范式。用生成式模型的架构做序列的建模。
