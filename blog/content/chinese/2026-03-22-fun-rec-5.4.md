# 5.4 推荐中 tokenize 的技术 读书笔记

原文链接 [https://github.com/datawhalechina/fun-rec/blob/master/docs/chapter_5_gr_basic/4.codebook.html](https://github.com/datawhalechina/fun-rec/blob/master/docs/chapter_5_gr_basic/4.codebook.html)

推荐系统中物品的表示方式，是从传统ID到语义ID的范式演进。

# 5.4.1 token 范式引进

### 三种范式的技术特征

**1. 稀疏 ID 范式（sparse id based）**

- 指为每一个物品分配一个唯一的原子 ID。
- 优势：无碰撞特征，交互自由，工程成熟度高。
- 实际应用中的三大困境：
    1. **词表爆炸问题**。每个 ID 作为一个词，空间维度非常巨大。
    2. **存储和泛化能力的双重困境**。维护每个 ID 的 embedding 会浪费大量模型容量预算，原子 ID 之间完全正交，导致模型对新物品毫无泛化能力。
    3. **协同信号的严重依赖**。需要依赖海量用户交互来学习物品之间的相似性，基于统计的方法在数据稀疏情况下效果极度恶化。

**2. 文本 ID 范式（text id based）**

- 指使用语言模型生成一组词，将商品信息、商品属性信息转化为自然语言。
  - 可以是一个长序列或树状结构，表示商品属性的组织。
- 优势：
    - 保留属性信息。
    - 避免过长的文本序列。
    - 语义丰富，表达中自然包含了序列信息和物品属性信息。
    - **泛化能力强**，因为语言模型的知识可以理解并生成没有见过的物品的自然描述。
    - **可解释性强**，因为推荐的结果是人类可读的文本。

这个范式在实际应用中有两大问题：

1. **表示效率差**：一个商品可能需要 10 个或者上百个 token 才能完整描述，仍然会有大量的计算成本和系统信息密度稀疏的问题。
2. **Grounding 困难**：因为生成序列是文本的虚拟 token，那么怎么能够精确地映射回具体物品呢？这是一个很大的挑战。比如说，可能会生成一个文本对应多个商品，可能生成的提前结束导致无法对应原来的商品，也有可能是生成的文本完全对应一个不存在的商品，也就是无法归位（Grounding）的问题。曾经有人尝试过先使用语言模型生成文本，然后再通过计算文本和物品集中所有物品 embedding 的 L2 距离进行重排。这实际上是利用额外的检索模块来弥补生成式框架的不足，违背了端到端的初衷。

**3. 语义 ID 范式（Semantic ID Based）**

这种方法是对前两种范式的一种革命性的突破。它指的是将物品生成一个固定长度序列的离散 token，每个 token 是一个可控大小的语义范畴。
   - 比如将一个 NBA 篮球球星扣篮集锦视频，生成的序列是“体育-竞技-篮球-扣篮-集锦”，这样每个词表示的是一个语义概念。
  这样的话，整个词表大小可以控制在 4000~6000 个词。

语义 ID 范式有三大优势：

- **可控的计算开销**：无论物品库多大，构成它的基本语义都是有限的。这样使得端到端的可微运算开销完全可以控制。
- **天然的层次化**：语义 ID 不是一个扁平的结构，而是层次化的序列。这种 token 的表示，前面的 token 表示的是粗粒度的类别，后面的 token 表示细粒度类别。这种结构天然支持了前缀匹配，也就是说模型生成中先匹配大类，然后再逐渐细化。
- **从记忆到推理的跨越**：由于模型学会了对语义 ID 的生成，则相似性关系会存在于 token 序列的结构中。也就是说，所有篮球视频共享一个前缀，所有教学类的视频共享序列的某个后缀。一旦模型学会了对篮球的偏好，它就能自然地将其认知泛化到所有包含该语义部分的新物品中，即使这些物品从未在训练中出现。也就是说，相似性关系已经被 token 序列所学会。

### 传统 ID 的表示困境

传统的原子 ID 在判别式架构中运作良好，但是在生成式架构中则不兼容。
  - 因为传统 ID 的超大规模词表在生成式建模中，数学和工程上都不可行。

那么语义 ID 的设计哲学，是将身份识别变成一种语义描述。不再随机地通过分配数字的方式来表示物体，而是通过一串存在于语义空间的序列来表示物品的内容和属性。
它符合人类的认知运作方式，即层次化的概念组合。从大类“科幻”到小类“悬疑”、“AI 视觉效果”，**这种层次化的概念组合，天然编码了物品之间的相似性关系**。

### 语义 ID 的生成信号

在工程中，语义 ID 通过两种信号来生成：

1. **内容信号**：内容信息实际上就是用预训练模型（比如 CLIP 或者 BERT）来提供语义向量。
2. **协同信号**：物品和用户之间交互的矩阵，蕴含着群体行为模式。

通过离散化的编码方式，生成序列化的语义 ID。其优势在于：

1. **可控的固定词表**。
2. **天然的层次化结构**（纵向或横向）。这使得模型具有强大的归纳偏置，即基于前缀的泛化能力。
3. **从记忆到推理的跨越**。语义 ID 最深刻的价值在于它是一种概念层面的推理，而非传统的查表式记忆。传统的 ID 是一种查表式记忆，即模型必须对每个物品有足够的观察数据才能匹配。而语义 ID 支持推理，例如：
    - **一阶推理**：如果用户喜欢 A，则用户很有可能喜欢相同前缀的物品，即基于前缀的泛化。
    - **二阶推理**：当用户观看 A 时，倾向于点击另外一个类别 B，则模型会意识到某些前缀之间存在着跨类别的迁移模式。
    - **组合推理**：模型可以从不同的维度之间学习偏好之间的组合。比如说，用户喜欢教学类或者篮球，则用户很有可能喜欢教学篮球类视频。这种从记忆到推理的跨越，使得语义 ID 在数据稀疏场景下仍然保持非常好的性能，解决了主流的冷启动问题。
